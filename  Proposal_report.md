### 1.2 自制数据集

由于当前项目主要采用**手机多视角拍摄**进行数据采集，因此自制数据集将围绕手机图像获取展开，用于支撑“基于视觉的三维重建”任务在**传统方法版**和**深度学习方法版**中的实验验证、对比分析与结果展示。

#### 1.2.1 采集场景
自制数据集计划覆盖以下三类典型场景：

- **小型静态物体**：如手办、杯子、盒子、书本、背包、小型雕塑摆件等。
- **室内局部场景**：如桌面区域、书架一角、实验台、宿舍局部区域等。该类数据适合用于分析背景复杂、遮挡较多、纹理不均匀等因素对重建效果的影响。
- **室外静态目标**：如校园雕塑、教学楼入口、小型建筑立面、亭子、标志物等。该类数据适合测试较大尺度场景下的多视图重建能力。

#### 1.2.2 采集设备与拍摄要求
自制数据集主要采用**手机主摄像头**进行拍摄。为了保证数据一致性，计划尽量固定同一个镜头、同一种拍摄模式和统一分辨率，避免频繁切换焦距和模式。

为了提高后续三维重建的成功率，拍摄过程中计划遵循以下原则：

- 相邻图像之间保持较高重叠率，建议约 70%
- 尽量保持目标静止
- 尽量保持光照稳定
- 尽量避免运动模糊、过曝、欠曝
- 尽量避免大面积玻璃、镜面和强反光表面
- 尽量减少背景中的动态干扰，如行人、车辆、晃动树叶等

#### 1.2.3 采集方式与规模
对于小型目标，计划采用手机围绕目标进行环绕拍摄的方式（或从视频截取），从平视、略微俯视和略微仰视等不同高度采集多组视角，每组拍摄 **30～80 张图像**。

对于室内局部场景，计划沿目标周围缓慢移动拍摄，保证相邻图像视角差不要过大，5～8 个对象，每个对象 30～60 张图像

对于室外雕塑或建筑立面，计划从多个方向进行拍摄，尽量覆盖正面、侧面及局部细节，3～5 个场景，每个场景 40～80 张图像


## 2. 传统方案计划

### 2.1 方案概述
本方案面向手机拍照场景下的单目标静态三维重建任务，适用于桌面物体、雕像、小摆件、盒子、玩具等目标。  
采集方式为使用手机围绕目标进行多角度拍摄，利用传统计算机视觉方法完成从二维图像到三维结构的恢复。  
整体流程包括：图像采集、相机标定、图像去畸变、特征提取与匹配、两视图初始化、增量式 SfM 重建、Bundle Adjustment 优化，以及可选的稠密重建。  
最终输出目标的稀疏点云，并进一步生成稠密点云或网格模型。

---

### 2.2 技术流程

#### 2.2.1 图像采集
使用手机对目标进行多视角拍摄，形成输入图像序列。  
该阶段输出为原始图像集合。

#### 2.2.2 相机标定
用现有的光学标定版，用手机拍摄多张不同角度的标定图像，利用 OpenCV 标定方法估计相机内参矩阵和畸变参数。  
该阶段输出包括：

- 相机内参矩阵
- 畸变系数
- 重投影误差评估结果

#### 2.2.3 图像去畸变
根据标定结果，对原始图像进行去畸变预处理，减小镜头畸变对特征匹配与几何估计的影响。  
该阶段输出为去畸变后的图像集合。

#### 2.2.4 特征提取
使用**SIFT**在每张图像中提取局部特征点及其描述子。  
该阶段输出包括：

- 每张图像中的关键点
- 对应的特征描述子

#### 2.2.5 特征匹配
对图像对之间进行特征匹配，并结合最近邻比值检验和交叉验证去除明显错误匹配。  
随后利用 RANSAC 估计基础矩阵或本质矩阵，进一步剔除外点。  
该阶段输出为高质量匹配点对。

#### 2.2.6 两视图初始化
从重叠率较高的一对图像开始，基于匹配点计算本质矩阵，恢复两相机之间的相对位姿，并通过三角化获得初始三维点。  
该阶段输出包括：

- 初始两视图相机位姿
- 初始稀疏三维点云

#### 2.2.7 增量式 SfM 重建
在初始重建基础上，逐步加入新的图像。  
对每张新图像，通过 PnP 方法估计其相机位姿，再结合已有图像进行三角化，恢复新的三维点。  
该过程不断迭代，逐步构建完整的稀疏点云和相机轨迹。  
该阶段输出包括：

- 多视图相机位姿
- 完整稀疏点云
- 相机运动轨迹

#### 2.2.8 Bundle Adjustment 优化
对已恢复的相机参数和三维点进行全局非线性优化，最小化重投影误差，提高整体重建精度和稳定性。  
该阶段输出为优化后的相机参数和稀疏点云。

#### 2.2.9 可选：稠密重建
在稀疏 SfM 结果基础上，进一步采用多视图立体（MVS）或简化版深度估计方法生成稠密点云。  
如条件允许，还可进一步进行表面重建，输出三维网格模型。  
该阶段输出包括：

- 稠密点云
- 三维网格模型

---

### 2.3 最终输出结果
本方案最终可输出以下结果：

- 相机标定参数
- 相机位姿估计结果
- 稀疏三维点云
- 稠密点云（可选）
- 网格模型（可选）
- 重建过程可视化结果

## 3. 深度学习方案计划

本项目深度学习方法版拟采用“**DUSt3R + LoRA 域适配 + Neuralangelo 高保真表面重建**”的技术路线，并设置 **COLMAP + 3D Gaussian Splatting** 作为备选方案。该方案面向手机多视角图像输入，目标是输出稠密点云、相机位姿、三维网格模型以及新视角渲染结果。

### 3.1 方案选择
主方案选择 **DUSt3R + LoRA + Neuralangelo**。其中，DUSt3R 用于从手机拍摄的多视角图像中恢复稠密几何信息和相机参数，解决传统方法中对精确标定和位姿初始化依赖较强的问题；Neuralangelo 用于在几何先验基础上完成高保真三维表面重建，输出更加完整和细致的三维模型。

备选方案选择 **COLMAP + 3D Gaussian Splatting**。该方案训练速度较快、实现较成熟、结果展示效果较好，适合作为可运行的补充方案或实验对照方案。

### 3.2 选择原因
选择该方案主要基于以下考虑：

- **适合手机多视角图像输入。** 手机采集的数据通常存在拍摄轨迹不规则、位姿不稳定、部分视角覆盖不足等问题。DUSt3R 对无标定或弱标定图像集合具有较好的适应性，更适合手机拍摄场景。
- **支持轻量化微调。** 可在 Transformer 注意力层中引入 LoRA，仅训练少量适配参数，降低显存占用，适合单张 RTX 5090 环境下训练。

### 3.3 训练方式与资源可行性
本项目深度学习方法版将在**单张 RTX 5090 显卡**上进行训练和实验。为了保证可行性，计划采用如下训练策略：

- 使用 **LoRA 微调**，避免全参数训练带来的显存压力
- 使用 **混合精度训练** 降低显存占用
- 使用 **gradient checkpointing** 进一步节省中间激活存储
- 先在较低分辨率下完成功能验证，再逐步提高输入分辨率
- 对 Neuralangelo 的重建分辨率和网格提取参数进行控制，平衡显存消耗与重建质量

综合来看，主方案在单张 RTX 5090 上具有较好的训练与运行可行性，备选方案则更适合作为训练更快、实现更稳的保底方案。

深度学习方法版预计输出以下结果：

- 稠密点云结果
- 相机位姿估计结果
- 三维网格模型
- 新视角渲染结果
- 与传统方法版的定性、定量对比分析结果
