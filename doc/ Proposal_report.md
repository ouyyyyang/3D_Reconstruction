# 基于视觉的三维重建
小组成员：欧阳卓涵，崔文伙，吴承臻，刘楷，蔡俊杰

## 1. 计划数据集
### 1.1 公开数据集

选择DTU数据集中的部分场景。[数据集下载链接](https://aistudio.baidu.com/datasetdetail/129222)

### 1.2 自制数据集

由于当前项目主要采用**手机多视角拍摄**进行数据采集，因此自制数据集将围绕手机图像获取展开，用于支撑“基于视觉的三维重建”任务在**传统方法版**和**深度学习方法版**中的实验验证、对比分析与结果展示。

#### 1.2.1 采集场景
自制数据集计划覆盖以下三类典型场景：

- **小型静态物体**：如手办、杯子、盒子、书本、背包、小型雕塑摆件等。
- **室内局部场景**：如桌面区域、书架一角、实验台、宿舍局部区域等。该类数据适合用于分析背景复杂、遮挡较多、纹理不均匀等因素对重建效果的影响。
- **室外静态目标**：如校园雕塑、教学楼入口、小型建筑立面、亭子、标志物等。该类数据适合测试较大尺度场景下的多视图重建能力。

#### 1.2.2 采集设备与拍摄要求
自制数据集主要采用**手机主摄像头**进行拍摄。为了保证数据一致性，计划尽量固定同一个镜头、同一种拍摄模式和统一分辨率，避免频繁切换焦距和模式。

为了提高后续三维重建的成功率，拍摄过程中计划遵循以下原则：

- 相邻图像之间保持较高重叠率，建议约 70%
- 尽量保持目标静止
- 尽量保持光照稳定
- 尽量避免运动模糊、过曝、欠曝
- 尽量避免大面积玻璃、镜面和强反光表面
- 尽量减少背景中的动态干扰，如行人、车辆、晃动树叶等

#### 1.2.3 采集方式与规模
对于小型目标，计划采用手机围绕目标进行环绕拍摄的方式（或从视频截取），从平视、略微俯视和略微仰视等不同高度采集多组视角，每组拍摄 **30～80 张图像**。

对于室内局部场景，计划沿目标周围缓慢移动拍摄，保证相邻图像视角差不要过大，5～8 个对象，每个对象 30～60 张图像

对于室外雕塑或建筑立面，计划从多个方向进行拍摄，尽量覆盖正面、侧面及局部细节，3～5 个场景，每个场景 40～80 张图像


## 2. 传统方案计划

### 2.1 方案概述
本项目传统方法基线采用成熟开源重建管线：**COLMAP + OpenMVS**。该路线避免重复造轮子，具有工程稳定性高、社区验证充分、结果可复现性好的优势。

### 2.2 技术流程

#### 2.2.1 SfM：COLMAP 稀疏重建与位姿估计
直接使用 COLMAP 完成特征提取、特征匹配、增量式 SfM 与 Bundle Adjustment。COLMAP 内置自标定流程可联合优化相机内参与畸变参数，并输出高精度相机位姿和稀疏点云。

该阶段输出包括：

- 相机内参与畸变参数
- 稀疏点云
- 多视图相机位姿（供后续学习方法使用）

#### 2.2.2 MVS：OpenMVS 稠密重建
将 COLMAP 的稀疏重建结果导入 OpenMVS，执行深度估计与多视图融合，生成稠密点云。

该阶段输出包括：

- 稠密点云
- 深度一致性更高的场景几何细节

#### 2.2.3 表面重建与纹理映射
继续使用 OpenMVS 的网格重建模块（Poisson Surface Reconstruction 或 Delaunay 三角化）提取三维网格，并进行纹理映射，获得可视化与评估友好的模型结果。

该阶段输出包括：

- 三维网格模型（Mesh）
- 纹理化网格（Textured Mesh）

### 2.3 最终输出结果
本方案最终可输出以下结果：

- 相机位姿与内参估计结果
- 稀疏点云与稠密点云
- 三维网格与纹理化网格
- 可用于对比实验的传统基线性能结果

## 3. 深度学习方案计划


### 3.1 方案选择


- **COLMAP + 2DGS 或 SuGaR**
	- 标准 3DGS 具备极快渲染速度，但直接网格提取质量通常较差。
	- 2DGS 与 SuGaR 侧重提升从 Gaussian 表达到高质量网格的可提取性，更契合“重建 + 网格输出”的任务目标。

- **备选方案：COLMAP + 原版 3DGS / Instant-NGP**
	- 当 2DGS/SuGaR 环境配置受阻时，使用原版 3DGS 或 Instant-NGP 保底。
	- 备选方案重点保障新视角渲染效果与训练速度；网格质量作为次要目标。

### 3.2 选择原因
选择依据如下：

- 与 COLMAP 位姿先验兼容，工程落地风险低。
- 相比重型神经隐式方法，训练周期更短，资源占用更可控。
- 同时满足“高质量新视角渲染”与“可提取网格”两类实验目标。

### 3.3 训练方式与资源可行性
为保障资源可行性，统一采用 720P 输入，配合 COLMAP 先验减少训练不稳定性。实验优先保证流程跑通与指标可复现，其次再做分辨率和训练轮次的增量优化。

### 3.4 技术流程设计

#### 3.4.1 数据预处理
将手机采集数据先送入 COLMAP，估计相机位姿并生成稀疏点云作为初始化先验。数据集按固定比例划分训练/测试集，建议抽取约 **1/8 图像**作为测试集，其余用于训练。

#### 3.4.2 模型训练
将训练图像与 COLMAP 位姿输入 2DGS（或 3DGS）模型，采用光度一致性目标进行优化，损失函数以 **L1 + D-SSIM** 为主，优化对象包括：

- 高斯中心位置
- 缩放参数
- 旋转参数
- 不透明度
- 球谐系数（SH）

#### 3.4.3 网格提取（Mesh Extraction）
在 2DGS/SuGaR 路线下，利用其几何表达特性执行官方或社区提供的 TSDF 融合/网格导出流程，生成高质量纹理网格，用于几何评测与可视化展示。

#### 3.4.4 结果评估与对比
深度学习方案将与传统基线在以下维度对比：

- 新视角渲染质量（PSNR / SSIM / LPIPS）
- 几何质量（在 DTU 等有真值数据上的几何误差）
- 训练与重建耗时
- 视觉效果（细节、完整性、纹理质量）

### 3.6 预期输出结果
深度学习方法版预计输出以下结果：

- 可实时渲染的 Gaussian 场景表示
- 新视角渲染结果与客观指标
- 可导出的三维网格及纹理模型
- 与 COLMAP + OpenMVS 基线的定性/定量对比结论
